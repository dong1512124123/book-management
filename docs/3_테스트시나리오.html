2인증 (회원가입 / 로그인 / 비밀번호 찾기)

TC-ID테스트 항목절차예상 결과결과비고
AU-01 회원가입 - 정상 로그인 화면 → 회원가입 → 이름, 전화번호, 아이디, 비밀번호 입력 → 가입하기 로그인 화면으로 이동, "회원가입 완료" 메시지 표시
AU-02 회원가입 - 중복 아이디 기존 아이디와 동일한 아이디로 가입 시도 에러 메시지 표시, 가입 실패
AU-03 회원가입 - 필수값 누락 이름 또는 비밀번호 등을 비워두고 가입 시도 필수 입력 안내 메시지, 가입 불가
AU-04 전화번호 자동 하이픈 회원가입 시 전화번호에 숫자만 입력 (01012345678) 자동으로 010-1234-5678 형식으로 변환
AU-05 관리자 로그인 아이디: admin, 비밀번호: 1234로 로그인 관리자 대시보드(/dashboard)로 이동
AU-06 관리자 최초 로그인 - 정보 입력 초기 admin 계정으로 첫 로그인 관리자 정보 설정 화면으로 리다이렉트 (이름, 전화번호 입력)
AU-07 사용자 로그인 AU-01에서 가입한 계정으로 로그인 사용자 대시보드(/user/dashboard)로 이동
AU-08 로그인 실패 잘못된 아이디 또는 비밀번호 입력 "아이디 또는 비밀번호가 올바르지 않습니다" 에러 표시
AU-09 로그아웃 로그인 후 상단의 로그아웃 클릭 로그인 화면으로 이동, "로그아웃되었습니다" 메시지 표시
AU-10 비밀번호 찾기 - 정상 이름, 아이디, 전화번호 올바르게 입력 → 비밀번호 초기화 비밀번호가 1234로 초기화, 성공 메시지 표시
AU-11 비밀번호 찾기 - 정보 불일치 이름 또는 전화번호가 틀린 상태로 초기화 시도 에러 메시지 표시, 초기화 실패

3관리자 - 대시보드

TC-ID테스트 항목절차예상 결과결과비고
AD-01 대시보드 통계 표시 관리자 로그인 후 대시보드 확인 전체 도서, 전체 회원, 대출 중, 연체 건수 정확히 표시
AD-02 승인 대기 알림 배너 사용자가 대출 신청한 상태에서 대시보드 확인 "승인 대기 중인 대출 신청이 N건 있습니다" 배너 표시
AD-03 메가 메뉴 동작 상단 메뉴에 마우스 호버 하위 메뉴(공지사항, 도서관리, 회원관리 등) 드롭다운 표시
AD-04 대시보드 통계 카드 클릭 각 통계 카드(전체 도서, 등록 회원, 대출 중, 대출 가능) 클릭 클릭한 카드에 해당하는 목록 페이지로 이동 확인

4관리자 - 공지사항 관리

TC-ID테스트 항목절차예상 결과결과비고
NO-01 공지 작성 공지사항 → 공지 작성 → 제목, 내용 입력 → 등록 공지 목록에 새 공지 표시, 성공 메시지
NO-02 공지 작성 시 알림 발송 NO-01 수행 후, 사용자 계정으로 알림 확인 모든 사용자에게 "새 공지사항" 알림 발송됨
NO-03 공지 상세 보기 공지 목록에서 제목 클릭 제목, 내용, 작성자, 작성일 표시
NO-04 공지 수정 공지 상세 → 수정 → 내용 변경 → 저장 변경된 내용 반영, 수정일 표시
NO-05 공지 삭제 공지 목록에서 삭제 클릭 → 확인 팝업 → 확인 공지 삭제, 목록에서 제거

5관리자 - 도서 관리

TC-ID테스트 항목절차예상 결과결과비고
BK-01 도서 등록 도서 관리 → 도서 등록 → 제목, 저자, 출판사, ISBN 입력 → 등록 도서 목록에 새 도서 추가, 성공 메시지
BK-02 도서 목록 조회 도서 관리 → 도서 목록 전체 도서 목록 테이블 표시 (번호, 제목, 저자, 출판사, ISBN, 상태)
BK-03 도서 검색 (전체) 검색 유형 "전체" → 키워드 입력 → 검색 제목, 저자, 출판사에서 키워드를 포함하는 도서만 표시
BK-04 도서 검색 (제목/저자/출판사) 각 유형 선택 후 검색 해당 항목에서만 검색 결과 표시
BK-05 도서 상세 보기 도서 제목 클릭 도서 정보 + 대여 정보(대여 중인 경우) 표시
BK-06 도서 수정 도서 상세 → 수정 → 정보 변경 → 저장 변경된 정보 반영, 성공 메시지
BK-07 도서 삭제 (비치 중) 대출 중이 아닌 도서 삭제 도서 삭제 완료
BK-08 도서 삭제 (대출 중) 대출 중인 도서 삭제 시도 삭제 불가 에러 메시지 표시
BK-09 도서 목록 다운로드 도서 목록 → 도서 목록 다운로드 클릭 book-list.pdf 파일 다운로드, 전체 도서 목록 포함
BK-10 검색 후 다운로드 도서 검색 → 도서 목록 다운로드 검색된 도서만 PDF에 포함
BK-11 뷰 전환 (목록/카드) 목록형/카드형 버튼 클릭 교차 테이블 ↔ 카드 뷰 전환, 새로고침 후에도 유지
BK-12 도서 목록 정렬 도서 목록에서 제목/저자/출판사/ISBN/상태 헤더 클릭 클릭한 헤더 기준 오름차순/내림차순 정렬 확인
BK-13 도서 삭제 (과거 대출 이력 있음) 반납 완료된 대출 이력이 있는 도서 삭제 도서와 관련 대출 이력 모두 삭제 완료
BK-14 도서 제목 말줄임 긴 제목의 도서를 등록 후 목록 확인 제목이 한 줄로 표시되고 넘치면 ...으로 말줄임

MB-02 회원 검색 이름 또는 전화번호로 검색 검색 조건에 맞는 회원만 표시
MB-03 회원 상세 보기 회원 이름 클릭 회원 상세 정보 표시
MB-04 회원 정보 수정 회원 상세 → 수정 → 정보 변경 → 저장 변경 사항 반영
MB-05 회원 삭제 회원 목록에서 삭제 클릭 회원 삭제 완료
MB-05a 회원 삭제 (과거 대출 이력 있음) 반납 완료된 대출 이력이 있는 회원 삭제 회원과 관련 대출 이력 모두 삭제 완료
MB-06 회원 목록 정렬 회원 목록에서 이름/전화번호/등록일 헤더 클릭 클릭한 헤더 기준 오름차순/내림차순 정렬 확인

7관리자 - 대출/반납 관리

TC-ID테스트 항목절차예상 결과결과비고
LN-01 대출 목록 - 전체 탭 대출 관리 → 대출/반납 목록 → 전체 탭 모든 대출 건이 표시됨
LN-02 대출 목록 - 상태별 탭 필터 승인대기 / 대출중 / 반납대기 / 반납완료 탭 각각 클릭 해당 상태의 대출 건만 필터링 표시
LN-03 대출 승인 승인대기 목록에서 승인 클릭 상태가 APPROVED로 변경, 사용자에게 알림 발송
LN-04 대출 거절 승인대기 목록에서 거절 클릭 대출 건 삭제, 도서 상태 비치 중으로 복원
LN-05 반납 확인 반납대기 목록에서 반납확인 클릭 상태 RETURNED, 반납일 기록, 사용자에게 알림 발송
LN-06 연체 표시 반납기한 경과 후 대출중 도서 확인 연체 뱃지 표시, 대시보드 연체 카운트 반영
LN-07 대출 목록 정렬 대출 목록에서 도서명/저자/출판사/회원명/대출일/반납기한/반납일/상태 헤더 클릭 클릭한 헤더 기준 오름차순/내림차순 정렬 확인
LN-08 대출 목록 도서명 말줄임 긴 도서명의 대출 건 확인 도서명이 최대 두 줄까지 표시되고 넘치면 ...으로 말줄임

UL-03 내 대출 현황 조회 내 대출 메뉴 클릭 본인의 대출 기록만 표시 (도서명, 저자, 대출일, 반납기한, 상태)
UL-04 반납 신청 대출 중인 도서 → 반납 버튼 → 확인 상태가 반납 대기로 변경, 관리자에게 알림 발송
UL-05 대출현황에서 상세 이동 내 대출에서 도서 제목 클릭 도서 상세로 이동, "← 대출 현황으로" 버튼 표시
UL-06 상세에서 대출현황 복귀 도서 상세에서 "← 대출 현황으로" 클릭 내 대출 현황 페이지로 정상 복귀

16통합 시나리오 (E2E)

아래는 전체 기능을 연결하는 종합 테스트 시나리오입니다.

시나리오 A: 신규 사용자의 도서 대출 ~ 반납 전체 흐름

순서역할수행 동작확인 사항
1사용자회원가입 (이름: 테스트유저, 아이디: testuser, 비밀번호: test1234)가입 성공 → 로그인 화면 이동
2사용자로그인 (testuser / test1234)사용자 대시보드 표시
3사용자도서 목록에서 비치 중인 도서 대출 신청"대출 신청 완료" 메시지
4관리자관리자 로그인 → 알림 확인"새 대출 신청" 알림 수신
5관리자대출/반납 목록 → 승인 클릭상태: 승인 대기 → 대출 중
6사용자알림 확인"대출 승인" 알림 수신
7사용자내 대출 현황에서 반납 신청상태: 대출 중 → 반납 대기
8관리자알림 확인 → 대출/반납 목록 → 반납확인 클릭상태: 반납 대기 → 반납 완료
9사용자알림 확인"반납 확인" 알림 수신
10사용자내 대출 현황 확인반납 완료 상태, 반납일 기록됨

시나리오 B: 공지사항 등록 및 확인 흐름

순서역할수행 동작확인 사항
1관리자공지사항 → 공지 작성 → 제목/내용 입력 → 등록공지 등록 성공
2사용자알림 확인"새 공지사항" 알림 수신, "공지사항 보기" 버튼 존재
3사용자"공지사항 보기" 클릭공지사항 목록으로 이동
4사용자해당 공지 제목 클릭공지 상세 내용 확인 가능

시나리오 C: 비밀번호 분실 및 복구

순서역할수행 동작확인 사항
1사용자로그인 화면 → 비밀번호 찾기비밀번호 찾기 화면 표시
2사용자이름, 아이디, 전화번호 입력 → 비밀번호 초기화"비밀번호가 1234로 초기화" 메시지
3사용자아이디 / 1234로 로그인로그인 성공
4사용자내 정보 수정에서 비밀번호 변경새 비밀번호로 로그인 성공
