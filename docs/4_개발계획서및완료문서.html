1프로젝트 개요

항목내용
프로젝트명LibraryGO - 간이 도서 대출 관리 시스템
목적소규모 도서관(학교, 학급, 동아리 등)에서 도서 등록, 대출/반납을 간편하게 관리하기 위한 웹 기반 시스템
대상 사용자관리자(도서 관리 담당자), 사용자(일반 회원)
버전v3.2 (인증, 대출 워크플로우, 알림, PDF, 삭제 개선 포함)
개발 기간2026년 2월
개발 도구Eclipse IDE + Spring Boot

2개발 환경

구분기술/도구버전
언어Java17
프레임워크Spring Boot3.2.3
빌드 도구Maven-
보안Spring Security6.x
ORMSpring Data JPA (Hibernate)-
템플릿 엔진Thymeleaf-

5데이터베이스 설계

5-1. admin (관리자)

컬럼타입설명
idBIGINT (PK, AUTO)관리자 ID
usernameVARCHAR(50), UNIQUE아이디
passwordVARCHAR(255)비밀번호 (BCrypt)
nameVARCHAR(50)이름
phoneVARCHAR(20)전화번호
first_loginBOOLEAN최초 로그인 여부
created_atDATETIME생성일

5-2. member (회원/사용자)

컬럼타입설명
idBIGINT (PK, AUTO)회원 ID
nameVARCHAR(50)이름
phoneVARCHAR(20)전화번호
usernameVARCHAR(50), UNIQUE아이디
passwordVARCHAR(255)비밀번호 (BCrypt)
roleVARCHAR(20)역할 (USER)
enabledBOOLEAN활성 상태
created_atDATETIME가입일

5-3. book (도서)

컬럼타입설명
idBIGINT (PK, AUTO)도서 ID
titleVARCHAR(200)제목
authorVARCHAR(100)저자
publisherVARCHAR(100)출판사
isbnVARCHAR(20)ISBN
descriptionTEXT도서 개요
created_atDATETIME등록일

5-4. loan (대출)

컬럼타입설명
idBIGINT (PK, AUTO)대출 ID
book_idBIGINT (FK)도서 ID
member_idBIGINT (FK)회원 ID
borrow_dateDATE대출일
return_due_dateDATE반납 예정일
returned_dateDATE실제 반납일
statusVARCHAR(30)REQUESTED / APPROVED / RETURN_REQUESTED / RETURNED

5-5. notice (공지사항)

컬럼타입설명
idBIGINT (PK, AUTO)공지 ID
titleVARCHAR(200)제목
contentTEXT내용
authorVARCHAR(50)작성자
created_atDATETIME작성일
updated_atDATETIME수정일

5-6. notification (알림)

컬럼타입설명
idBIGINT (PK, AUTO)알림 ID
recipient_typeVARCHAR(20)수신자 유형 (ADMIN/USER)
recipient_idBIGINT수신자 ID
typeVARCHAR(50)알림 유형 (LOAN_REQUESTED 등)
titleVARCHAR(200)제목
messageVARCHAR(500)내용
is_readBOOLEAN읽음 여부
created_atDATETIME생성일

5-7. login_history (로그인 이력)

컬럼타입설명
idBIGINT (PK, AUTO)이력 ID
user_typeVARCHAR(20)사용자 유형 (ADMIN/USER)
user_idBIGINT사용자 ID
usernameVARCHAR(50)로그인 아이디
login_timeDATETIME로그인 시각
ip_addressVARCHAR(50)접속 IP

7개발 일정 (타임라인)

1단계
기본 CRUD (도서/회원/대출)
도서 등록·목록·검색, 회원 관리, 대출/반납 기본 처리, Bootstrap UI 구축
2단계
인증 시스템
Spring Security 기반 로그인, 회원가입, 관리자/사용자 역할 분리, 비밀번호 찾기, BCrypt 암호화
3단계
대출 승인 워크플로우
사용자 대출 신청 → 관리자 승인 → 반납 신청 → 반납 확인, 상태별 탭 필터
4단계
알림 시스템
대출/반납/공지 알림 자동 발송, 읽음 처리, 알림 뱃지, 바로가기 버튼
5단계
이력 관리 + UI 개선
로그인 이력 자동 기록, 목록형/카드형 전환, 메가 메뉴, 공지사항 관리
6단계
PDF + 도서 상세 + 마무리
OpenPDF 도서 목록 PDF, 도서 상세 대여정보/마스킹, 검색 PDF 연동, 문서 작성
7단계
UI 개편 + 기능 확장
GNB+LNB 네비게이션 구조 전환, 도서 개요(description) 필드 추가, 테이블 정렬 기능, 대출 이력/로그인 이력 PDF 다운로드, 대시보드 통계 카드 바로가기
8단계
엑셀 다운로드 + 다운로드 UI 개선
Apache POI 엑셀(xlsx) 다운로드 기능 추가, 모든 다운로드 버튼 PDF/Excel 드롭다운 전환, 대출/반납 목록 다운로드 추가, 다운로드 버튼 카드 헤더 이동, 매뉴얼 바로가기 버튼

9보안 설계

보안 항목구현 방식
인증Spring Security FormLogin 기반 세션 인증
비밀번호BCryptPasswordEncoder로 해시 저장 (평문 저장 불가)
